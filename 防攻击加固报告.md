## 🔒 新增防护机制

### 1. ✅ 自我交易攻击防护

**攻击场景**: 用户可以将自己同时注册为算力提供者，然后自己给自己支付，从而套取平台资金。

**修复方案**:
```rust
// 防止用户和提供者是同一人
require!(
    ctx.accounts.user.key() != ctx.accounts.provider.key(),
    ErrorCode::SelfTransactionNotAllowed
);
```

**防护级别**: 🛡️🛡️🛡️🛡️🛡️ (5/5)

---

### 2. ✅ 垃圾交易攻击防护

**攻击场景**: 攻击者可以发送大量极小金额的交易，消耗网络资源和 gas。

**修复方案**:
```rust
// 新增最小 token 数限制
const MIN_TOKENS_PER_SUBMIT: u64 = 100;

require!(
    total_tokens >= MIN_TOKENS_PER_SUBMIT && total_tokens <= MAX_TOKENS_PER_SUBMIT,
    ErrorCode::InvalidTokenAmount
);
```

**防护级别**: 🛡️🛡️🛡️🛡️ (4/5)

---

### 3. ✅ 零成本攻击防护

**攻击场景**: 攻击者可以构造参数使得计算出的成本为 0，从而免费使用服务。

**修复方案**:
```rust
// 防止零成本攻击
require!(total_cost > 0, ErrorCode::InvalidCostAmount);

// 在 calc_cost 函数中
if tokens == 0 {
    return Ok(0);
}
```

**防护级别**: 🛡️🛡️🛡️🛡️🛡️ (5/5)

---

### 4. ✅ 余额不足攻击防护

**攻击场景**: 用户余额不足但仍然尝试交易，导致交易失败但消耗 gas。

**修复方案**:
```rust
// 检查用户余额是否足够（预留 5000 lamports 作为 gas）
let user_balance = ctx.accounts.user.to_account_info().lamports();
require!(
    user_balance >= total_cost.checked_add(5_000).ok_or(ErrorCode::CalculationOverflow)?,
    ErrorCode::InsufficientUserBalance
);
```

**防护级别**: 🛡️🛡️🛡️🛡️ (4/5)

---

### 5. ✅ 费用计算错误防护

**攻击场景**: 由于计算错误，分成总和可能超过总成本，导致资金损失。

**修复方案**:
```rust
// 验证分成总和不超过总成本
require!(
    dev_fee.checked_add(provider_fee).ok_or(ErrorCode::CalculationOverflow)? <= total_cost,
    ErrorCode::InvalidFeeCalculation
);
```

**防护级别**: 🛡️🛡️🛡️🛡️🛡️ (5/5)

---

### 6. ✅ 平台账户耗尽攻击防护

**攻击场景**: 攻击者可以通过大量提现操作耗尽平台账户余额，导致合约无法正常运行。

**修复方案**:
```rust
// 新增最小平台余额常量
const MIN_PLATFORM_BALANCE: u64 = 100_000_000; // 0.1 SOL

// 确保平台账户保留最小余额
require!(
    platform_balance >= earnings.checked_add(MIN_PLATFORM_BALANCE).ok_or(ErrorCode::CalculationOverflow)?,
    ErrorCode::InsufficientPlatformBalance
);
```

**防护级别**: 🛡️🛡️🛡️🛡️🛡️ (5/5)

---

### 7. ✅ 汇率操纵攻击防护

**攻击场景**: 
- 管理员恶意频繁更新汇率
- 管理员一次性大幅度修改汇率（如从 1:1 改为 1:1000）

**修复方案**:
```rust
// 新增汇率保护常量
const MAX_RATE_CHANGE_PERCENT: u64 = 50;      // 单次最多变化 50%
const MIN_RATE_UPDATE_INTERVAL: i64 = 300;    // 最小更新间隔 5 分钟

// 防止频繁更新
require!(
    clock.unix_timestamp >= platform_account.last_rate_update + MIN_RATE_UPDATE_INTERVAL,
    ErrorCode::RateUpdateTooFrequent
);

// 防止剧烈变化（最多 ±50%）
let max_rate = old_rate.checked_mul(150).ok_or(...)?.checked_div(100).ok_or(...)?;
let min_rate = old_rate.checked_mul(50).ok_or(...)?.checked_div(100).ok_or(...)?;

require!(
    lamports_per_yuan >= min_rate && lamports_per_yuan <= max_rate,
    ErrorCode::RateChangeTooLarge
);
```

**防护级别**: 🛡️🛡️🛡️🛡️🛡️ (5/5)

## 🔧 其他安全改进

### 1. 使用直接 lamports 操作替代 CPI

**原因**: CPI 调用可能被恶意合约拦截或操纵。

**修改**:
```rust
// 修改前：使用 CPI
let cpi_context = CpiContext::new(...);
anchor_lang::system_program::transfer(cpi_context, amount)?;

// 修改后：直接操作 lamports
**from_account.try_borrow_mut_lamports()? = from_account
    .lamports()
    .checked_sub(amount)
    .ok_or(ErrorCode::CalculationOverflow)?;

**to_account.try_borrow_mut_lamports()? = to_account
    .lamports()
    .checked_add(amount)
    .ok_or(ErrorCode::CalculationOverflow)?;
```

**优势**:
- 更安全，避免 CPI 攻击
- Gas 消耗更低
- 更直接的控制

---

### 2. 强化 CEI 模式

**Checks-Effects-Interactions 模式**:

```rust
// ✅ 正确顺序
// 1. Checks - 所有验证
require!(provider_account.is_provider, ErrorCode::NotProvider);
require!(provider_account.pending_revenue > 0, ErrorCode::NoEarnings);

// 2. Effects - 更新状态
provider_account.pending_revenue = 0;
provider_account.withdrawn = provider_account.withdrawn.checked_add(earnings)?;
provider_account.last_withdraw_time = clock.unix_timestamp;

// 3. Interactions - 转账
**platform_account.try_borrow_mut_lamports()? -= earnings;
**user.try_borrow_mut_lamports()? += earnings;
```

---

### 3. 改进溢出检查

**所有 `saturating_*` 替换为 `checked_*`**:

```rust
// 修改前：可能静默溢出
let total = a.saturating_add(b).saturating_add(c);

// 修改后：溢出时报错
let total = a
    .checked_add(b)
    .ok_or(ErrorCode::CalculationOverflow)?
    .checked_add(c)
    .ok_or(ErrorCode::CalculationOverflow)?;
```

---

### 4. 增强类型安全

**calc_cost 函数改进**:

```rust
// 确保结果在 u64 范围内
let lamports = lamports_128
    .try_into()
    .map_err(|_| ErrorCode::CalculationOverflow)?;
```
